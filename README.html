# Recruit 101 — Frontend (Static, GitHub Pages-ready)

This is a **static frontend** for your Google Apps Script backend. Deploy it on GitHub Pages (or any static host) and point it at your published Web App URL.

## 1) Configure

Edit `config.js` and paste your deployed Apps Script Web App URL:

```js
window.APP_CONFIG = {
  API_BASE: "https://script.googleusercontent.com/…/exec"
};
```

> You can also use the `.../macros/s/.../exec` URL if you prefer.

## 2) Run locally

Just open `index.html` in a browser. (If CORS is restricted on your domain, test via GitHub Pages.)

## 3) Deploy to GitHub Pages

1. Create a new repo, e.g. `recruit101-frontend`.
2. Add these files, commit, push.
3. In **Settings → Pages**, set **Source** = `main` → `/ (root)`.
4. Wait for Pages to build, then open your site URL.

## 4) Features

- Filter by `Status` (server-side).
- Search by name, surname, agency, or cell (client-side).
- Pagination with configurable rows per page (default 15).
- **Bulk set status** for selected rows (applies per row via the backend endpoint).
- Click **WhatsApp** to open in a **new window** (no navigating away).
- Quick **Stats** dialog calling `/stats` action.
- Clean, responsive UI with keyboard-friendly note editing (press **Enter** to save).

## 5) Back-end endpoints used

- `GET ?action=recipients&status=…`
- `GET ?action=stats`
- `POST { action: "updateSingleStatus", payload: { rowNumber, newStatus } }`
- `POST { action: "updateNote", payload: { rowNumber, note } }`

## 6) Troubleshooting

- If clicking WhatsApp opens in the same tab, we still force `window.open(url, "_blank", "noopener,noreferrer")` for a **new tab/window**.
- Make sure your Apps Script is deployed as a Web App (**Execute as you**, **Anyone with the link**).
- If you have multiple tabs in the Sheet, you can pass `&gid=<tabId>` to the backend URL to target a specific tab (the backend already supports that).
